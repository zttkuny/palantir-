## 🏗️ 平台与主题分类

### 一、Palantir Foundry 平台（17 份）

Foundry 是 Palantir 的核心数据整合与分析平台，提供端到端的数据科学解决方案。

#### 1.1 数据接入与集成（3 份）

| 文档名称 | 核心内容 | 关键技术 |
|---------|---------|---------|
| **数据接入与同步指南** | 创建首个数据连接，涵盖文件系统、关系型数据库和 REST API 三种连接方式 | Egress Policy、Source/Sync 架构、SSL 配置、RID 引用 |
| **数据连接全栈指南** | 深入解析数据连接架构，包括直接连接 vs 代理（Agents）选择 | 机密管理器、出口策略、数据血缘追溯 |
| **数据集成实战指南** | 更新版教程，介绍源基外部转换（Source-based external transforms）新方法 | REST API 代码集成、Python 转换、计划任务调度 |

**关键洞察**：数据连接采用 **Source（源）** 和 **Sync（同步）** 的架构设计，支持一对多关系。安全机制通过 Egress Policy 控制，确保符合企业安全规范。

#### 1.2 数据管道构建（4 份）

| 文档名称 | 核心内容 | 关键技术 |
|---------|---------|---------|
| **数据管道构建实战全指南** | 从零构建完整数据管道，包括数据清洗、集成、聚合和部署 | Pipeline Builder、数据分层（Raw/Clean/Enriched/Ontology）、增量转换 |
| **数据管道构建实战指南** (2份) | 基础管道构建实践，涵盖多格式数据处理 | Trim Whitespace、类型转换、结构体展平、AIP 助手 |
| **无代码数据管道实战手册** | 重点展示无代码方式构建生产级管道 | 多格式接入（CSV/Parquet/JSON）、AI 辅助转换、数据预期（Data Expectations） |

**关键洞察**：Pipeline Builder 支持 **低代码/无代码** 构建，通过 AI 助手（AIP Assist）可自动生成复杂转换逻辑。管道分层设计（Raw → Clean → Enriched → Ontology）提升可维护性。

#### 1.3 AI/LLM 集成（2 份）

| 文档名称 | 核心内容 | 关键技术 |
|---------|---------|---------|
| **RAG 工作流：PDF 预处理全指南** | 为 RAG 架构准备 PDF 文档的端到端流程 | 文本提取（Raw Text/OCR/Layout Aware）、分块（Chunking 512 tokens）、向量嵌入（Embeddings）、本体集成 |
| **管道构建器：LLM 实体提取实战指南** | 使用 LLM 从 PDF 中自动提取实体信息 | Use LLM 节点、实体提取模板、结构化输出解析（Extract many struct fields） |

**关键洞察**：RAG 工作流的核心是 **文本分块 + 向量嵌入 + 本体建模**。LLM 实体提取可直接处理 Media Set，无需预先提取文本，大大简化流程。

#### 1.4 数据分析（1 份）

| 文档名称 | 核心内容 | 关键技术 |
|---------|---------|---------|
| **数据分析实战全解** | Contour 分析全流程，从数据清洗到交互式看板 | Paths & Boards、表达式板（SQL 逻辑）、参数化过滤、数据血缘 |

**关键洞察**：Contour 采用 **路径（Paths）** 和 **板（Boards）** 的线性逻辑，支持路径依赖和全局过滤。

#### 1.5 应用开发（4 份）

| 文档名称 | 核心内容 | 关键技术 |
|---------|---------|---------|
| **应用开发深度实战指南** | Workshop 应用构建器，构建运营级应用 | 变量驱动逻辑、Marketplace 安装、Actions & Functions、TypeScript 函数 |
| **应用开发进阶指南** | 应用构建进阶技巧，强调运营闭环 | 对象集变量、变量链接、甘特图、条件显示 |
| **应用构建深度实战指南** | 从本体到应用的完整转换 | 页面布局、容器格式、条件可见性 |
| **代码存储库深度实践指南** | Pro-code 开发方式，使用 PySpark | Code Repositories、Git 集成、分支保护、数据血缘 |

**关键洞察**：Workshop 应用由 **本体（Ontology）** 驱动，通过变量（Variables）控制数据流。支持 Actions（本体操作）实现运营闭环，必要时可用 TypeScript 函数扩展逻辑。

#### 1.6 本体建模（2 份）

| 文档名称 | 核心内容 | 关键技术 |

| **本体建模实战指南** | 构建首个本体模型，理解对象、链接和关系 | 对象类型（Object Types）、主键/标题键、链接类型（1:1/1:Many/Many:Many）、连接表 |
| **本体函数实战速成指南** | 通过 TypeScript 函数扩展本体功能 | @Function、@OntologyEditFunction 装饰器、指标计算、函数驱动的图表和操作 |

**关键洞察**：本体是 Foundry 的**数字孪生**基础。链接关系支持复杂业务建模，函数可扩展本体能力，实现计算逻辑与数据修改的集成。

#### 1.7 数据治理与保护（1 份）

| 文档名称 | 核心内容 | 关键技术 |
|---------|---------|---------|
| **数据保护与治理实战指南** | 三大数据保护工具：Checkpoints、Sensitive Data Scanner、Cipher | 理由驱动的行为管控、敏感数据扫描（RegEx）、加密与脱敏、Cipher Channel/License |

**关键洞察**：数据保护采用**检测 → 约束 → 加密**的完整链路。Cipher 支持可逆加密和不可逆混淆，通过许可证机制控制访问。

### 二、Palantir Contour 平台（2 份）

Contour 是 Foundry 的低代码分析工具，专注于数据集（Data Sets）的分析和可视化。

| 文档名称 | 核心内容 | 关键技术 |
|---------|---------|---------|
| **高级报表与仪表板构建指南** | 复刻 Power BI 复杂仪表板，实现功能对等 | 表达式板（SQL 片段）、开窗函数（RANK）、分箱分段（Bucketing）、图表叠加（Overlays） |
| **工业级数据分析全流程实战指南** | 从预处理到交互式看板的完整分析流程 | 路径依赖、参数化过滤、直方图交叉过滤、Notepad 集成 |

**关键洞察**：Contour 通过 **Paths & Boards** 架构实现复杂分析。高级功能包括 SQL 表达式、窗口函数和参数化交互，可媲美专业 BI 工具。

### 三、Palantir Quiver 平台（2 份）

Quiver 专门用于分析本体（Ontology）对象，特别适合处理时间序列和大规模对象数据。

| 文档名称 | 核心内容 | 关键技术 |
|---------|---------|---------|
| **本体数据分析实战指南** | Quiver 基础与进阶功能，构建可复用分析 | 卡片（Cards）、画布/图表视图、转换表（Transform Tables）、视觉函数（Visual Functions） |
| **深度解析：本体数据分析全指南** | 完整分析流程，从数据准备到高级可视化 | 对象连接、公式计算、Vega 图表、跨应用集成 |

**关键洞察**：Quiver 的 **视觉函数** 可将分析逻辑封装为可复用组件。转换表操作上限为 50,000 行，超过需使用物化（Materializations）技术。

AIP（人工智能平台）集成 AI 能力，构建端到端智能工作流。

| 文档名称 | 核心内容 | 关键技术 |
|---------|---------|---------|
| **AIP 实战：构建端到端智能工作流** | 将 PDF 转化为知识图谱，构建文献综述应用 | PDF 处理、分块、嵌入、OAG（本体增强生成）、Vertex 知识图谱 |

## 💡 关键发现与洞察

### 1. 架构设计理念

- **本体驱动**：Foundry 以本体（Ontology）为核心，将数据转化为业务对象，支撑上层应用
- **低代码优先**：Pipeline Builder、Contour、Workshop 等工具支持低代码/无代码开发
- **代码可选**：Code Repositories 提供 Pro-code 选项，满足高级需求
- **数据血缘**：全程维护数据出处，确保可追溯性和可重现性

### 2. AI 集成策略

- **RAG 架构**：PDF 预处理 → 分块 → 嵌入 → 本体集成 → AIP 应用
- **OAG 模式**：本体增强生成，将 LLM 与受治理数据结合，避免幻觉
- **智能体开发**：从问答式 AI 到可执行操作的智能体，实现人机协作

### 3. 工具选择标准

| 场景 | 推荐工具 | 原因 |
|------|---------|------|
| 处理数据集（Data Sets） | Contour | 低代码分析，支持 SQL 表达式 |
| 处理本体对象 | Quiver | 专为对象设计，支持大规模数据 |
| 构建运营应用 | Workshop | 变量驱动，支持 Actions 操作 |
| 数据管道构建 | Pipeline Builder | 无代码/低代码，AI 辅助 |
| 复杂逻辑开发 | Code Repositories | Pro-code，PySpark/TypeScript |

### 4. 最佳实践建议

- **管道分层**：Raw → Clean → Enriched → Ontology，提升可维护性
- **数据预期**：设置 Data Expectations，主动监控数据质量
- **增量转换**：大规模数据使用增量处理，提升性能
- **视觉函数**：Quiver 中封装可复用逻辑，提高协作效率
- **评估套件**：AI 应用使用评估套件验证准确率，建立团队信任
